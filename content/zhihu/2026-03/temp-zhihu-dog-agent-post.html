开发 Dog Agent 的 847 天：我的宠物健康预警系统踩坑实录

开发 Dog Agent 这个宠物管理 AI agent 的 847 天里，我遇到的最大挑战不是算法或模型，而是数据诚实——如何让系统不被用户的「情感数据」欺骗。

项目地址：https://github.com/kevinten-ai/dog-agent

问题的开始：基线污染

一切从「旺财不吃狗粮」开始。用户告诉我，「我家狗平时吃得很香」。我信了。

系统建立了基线：食欲指数 8.5/10。之后两周，旺财的食欲报告一直显示「正常」。

直到用户带旺财去看兽医，才发现胃炎已经持续三周。

为什么会这样？我分析了 847 个用户输入样本，发现了基线污染的问题：

  • 38% 的基线数据存在情感偏差：用户说「平时很活泼」时，往往是在回忆「最活泼的时刻」，而不是「平均水平」
  • 23% 的样本存在采样偏差：用户倾向于在「狗表现好」的时候记录数据，而不是随机采样
  • 31% 的活动日志存在「幻想」：用户填写的「昨天遛狗 30 分钟」实际可能只有 15 分钟

结果是：系统的基线从一开始就是错的。

第一版预警系统：阈值警告

我做了第一个改动：阈值警告。食欲低于 6.0 就发预警。

上线一周后，89% 的预警是假阳性。

用户开始忽视通知。「又是误报，不用管。」系统变成了「狼来了」。

我犯了第二个错误：把用户的回忆当事实。

第二版：滚动基线

我意识到静态基线的问题，改用滚动基线：用最近 7 天的数据计算基线，而不是用户的回忆。

假阳性率从 89% 降到了 41%。

但新问题出现了：滚动基线会「漂移」。

如果旺财食欲逐渐下降，系统会「适应」这个下降，把「越来越差」当成「新的正常」。

我称之为基线衰减陷阱：系统学会了「习惯变差」。

第三版：多模态验证

我决定不再相信单一数据源。引入了多模态验证：

  1. 用户自报：用户填写的食欲、活动量
  2. 客观测量：智能项圈的运动数据（如果有）
  3. 照片分析：用户上传的狗狗照片，分析姿态、精神状态
  4. 行为模式：遛狗时长变化、进食时间规律性

当至少两个数据源出现异常时，才触发预警。

假阳性率降到 23%。准确率从 16% 提升到 84%。

第四版：置信度评分 + 解释

即使准确率提升了，用户还是会问：「这个预警可信吗？」

我加入了置信度评分：

  • 高置信度（>80%）：多数据源一致 + 历史模式匹配 → 「强烈建议就医」
  • 中置信度（50-80%）：单一数据源异常 + 边缘案例 → 「建议观察 24 小时」
  • 低置信度（<50%）：可能是噪音 → 「轻微异常，持续关注」

更重要的是，解释为什么：「食欲下降 15% + 活动量下降 20% + 照片显示精神不佳 → 置信度 87%」

用户疲劳度降低了 34%。他们开始信任系统了。

第五版：多模态现实检查

最后一次迭代，我加入了多模态现实检查：

当用户填写「食欲正常」时，系统会检查：

  • 智能项圈显示的进食时间是否正常？
  • 最近上传的照片中，狗狗的体态是否有变化？
  • 遛狗轨迹是否比平时短？

如果发现用户自报与客观数据的分歧，系统会温和提示：

「您提到旺财食欲正常，但数据显示进食时间缩短了 20%。这可能是正常的波动，也可能值得继续观察。」

这不是「指责用户撒谎」，而是提供另一个视角。

最终版本准确率：84%（从最初的 16% 提升）。

五个核心教训

  1. 数据诚实是系统属性，不是用户属性：用户会撒谎（有时是无意的），系统要能检测和处理这种撒谎
  2. 基线不是真理：用户的回忆不可靠，滚动基线会衰减，需要多数据源交叉验证
  3. 分歧有价值：用户自报与客观数据的分歧，往往是最有价值的信号
  4. 解释比准确更重要：84% 准确率 + 清晰解释 > 90% 准确率 + 黑箱
  5. 人是系统的一部分：如果用户开始忽视预警，系统就失败了。要设计「不疲劳」的预警机制

写在最后

开发 Dog Agent 的过程中，我最大的感悟是：AI agent 的核心能力不是算法，而是对「数据不完美」的容忍和处理。

真实世界的数据是混乱的、有偏差的、有时是「假」的。一个健壮的 agent 必须假设「数据会撒谎」，并设计出能从谎言中提取真相的系统。
