## 当前推荐标题模板

### 高效果模板 (已验证)
1. **How-to 型**: "How I {action} {number} Times and {result}"
   - 示例: "How I Deployed a Multi-Runtime SDK 847 Times..."
   - 效果: CTR 6.2%
   
2. **经验型**: "What I Learned From {experience}"
   - 示例: "What I Learned From 847 Days of Writing Open Source"
   - 效果: CTR 5.8%
   
3. **数字型**: "{Number} {Things/Lessons/Truths} About {topic}"
   - 示例: "5 Brutal Truths Nobody Tells You About..."
   - 效果: CTR 5.5%

4. **反直觉型**: "Why You Shouldn't {common practice}"
   - 示例: "Why You Shouldn't Build the Perfect AI Travel Agent"
   - 效果: CTR 5.1%

### 测试中模板
- "The {adjective} Guide to {topic}" (已测试1次, 待数据分析)
- "{Number} Days of {action}: What Happened" (已测试2次, 待数据分析)

### 当前使用优先级
1. 优先: How-to 型 (经验总结类)
2. 其次: 经验型 (教训总结类)
3. 备选: 数字型 (列表干货类)

## 当前推荐内容结构

### 结构A - 经验分享型 (Reaction rate: 4.5%)

```
1. Hook (50 words): Problem or surprising fact
2. Context (150 words): Why this matters
3. Story/Experience (500 words): Personal journey
4. Lessons/Challenges (600 words): 3-5 key points
5. Practical takeaways (200 words): Actionable advice
6. CTA (50 words): Question or discussion prompt
```

**字数**: 1500-1600 words  
**格式**: Markdown  
**适用**: 经验总结、项目复盘类

### 结构B - 技术教程型 (Bookmark rate: 6.2%)

```
1. Overview (100 words): What you'll learn
2. Problem statement (150 words): Why existing solutions fail
3. Solution (800 words): Step-by-step implementation
4. Code examples (300 words): Key snippets
5. Results/Comparison (150 words): Before/after
6. Next steps (100 words): Further reading
```

**字数**: 1600-1700 words  
**格式**: Markdown + Code blocks  
**适用**: 技术教程、实现方案类

## 优化建议摘要

### 已验证有效
- ✅ 具体数字支撑: "847 times" vs "many times"
- ✅ 失败经验: 比成功经验更吸引互动
- ✅ Strong CTA: 评论率 +12%
- ✅ Code examples: 收藏率 +15%

### 待验证
- ⏳ 发布时间优化 (考虑欧美时区)
- ⏳ Tags 组合效果
- ⏳ Cover image 影响

### Dev.to 特色
- ✅ Liquid tags: {% github %} 提升可信度
- ✅ Series: 连载增加粘性
- ✅ Canonical URL: 利于SEO

### 已禁用
- ❌ Clickbait 标题
- ❌ 与内容不符的标题
- ❌ 纯理论无实践的内容

## 跨平台复用策略

### 知乎 → Dev.to 适配
| 知乎内容类型 | Dev.to 适配方式 | 难度 |
|-------------|----------------|------|
| 经验总结类 | 直接翻译 + 调整 Hook | 低 |
| 技术架构类 | 重构为 Tutorial | 中 |
| 案例分析类 | 保持结构，调整语言 | 低 |

### 复用流程
1. 知乎发布 → 等待反馈
2. 高互动内容 → 筛选复用
3. 翻译重构 → 适配 Dev.to
4. 发布追踪 → 对比效果
