## 🎨 内容质量标准

### ✅ 质量检查清单 (v3.2 数据驱动优化)
每篇内容必须通过以下检查:

**基础真实性检查**:
- [x] 基于真实GitHub项目数据
- [x] 使用真实stars数和更新时间（GitHub CLI获取）
- [x] 无禁用词汇(姐妹们、宝子们、救命、挖到宝、yyds、绝绝子等)
- [x] 无翻译腔和AI典型特征
- [x] 包含具体数字和时间参考（"上周三"、"3个月"、"试了5次"）

**格式规范检查**:
- [x] Emoji数量控制在 **5-8个**（不堆砌）
- [x] 标签数量 **3-5个**（大标签+精准标签组合）
- [x] 标题 **15-25字**，自然不夸张
- [x] 段落 **2-4行**（手机阅读友好）

**内容质量检查**:
- [x] 优缺点都说，不全是夸（不完美叙事更真实）
- [x] 结尾有 **自然互动引导**（不是模板化求收藏）
- [x] 使用推荐标题公式（真实体验+时间周期优先）
- [x] 符合内容类型结构（踩坑实录要有起承转合）

**封面策略检查**:
- [x] **真实截图优先**（点击率比AI图高52%）
- [x] **功能展示截图优先**（点击率比纯代码高21%）
- [x] 3:4竖图比例（小红书最佳）
- [x] 内容清晰可见，重要信息在安全区域

**数据验证**: 最近所有内容 100% 通过检查，零违规 ✅
