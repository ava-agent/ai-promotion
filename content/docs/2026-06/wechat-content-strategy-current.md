## 二、标题策略

### ✅ 验证成功模板
```
用了{项目名}三个月，聊聊{核心主张}到底香不香
```

### 📚 扩展标题模板库

| 模板类型 | 模板公式 | 适用场景 | 预期点击率 |
|----------|----------|----------|------------|
| 项目体验型 (验证成功) | `用了{项目}三个月，{主张}` | 开源项目推广 | ⭐⭐⭐⭐⭐ |
| 踩坑实录型 | `踩了{数字}个坑，整理出开箱即用配置` | 部署指南、排障 | ⭐⭐⭐⭐⭐ |
| 反常识误区型 | `别再{错误做法}！{数字}%的人都错了` | 工具配置、常见错误 | ⭐⭐⭐⭐⭐ |
| 利益承诺型 | `这{数字}个技巧，让你{具体收益}` | 效率提升、工具技巧 | ⭐⭐⭐⭐⭐ |
| 问题悬念型 | `为什么{反常识现象}？{数字}个原因` | 原理分析、技术对比 | ⭐⭐⭐⭐ |

### 示例
- 《用了Capa-Java三个月，我差点被"一次编写，到处运行"忽悠瘸了》
- 《用了Capa-BFF三个月，聊聊零成本BFF到底香不香》
- 《MCP 生产环境 Docker 部署：我踩了 94 个坑后整理的开箱即用配置》

### 标题优化原则
1. **时间锚定**: "三个月" 暗示真实使用体验，不是随手评测
2. **口语化**: "聊聊"、"到底香不香" 拉近和读者距离
3. **争议性/好奇心**: "忽悠瘸了"、"香不香"、"踩了N个坑" 引发点击欲望
4. **包含项目名/关键词**: 有利于搜索和记忆

### 点击率预期优化
- 保持标题长度在20-30字之间
- 核心关键词前置
- 避免太技术化的术语堆彻
- 每篇文章选择最适合的模板，不强制统一

## 三、文章结构策略

### 标准结构（8段式）

| 段落 | 内容 | 字数建议 | 目的 |
|------|------|----------|------|
| 1. **开篇 hook** | 为什么会用上这个项目，当时的期待 | 150-250字 | 引发共鸣，吸引继续读 |
| 2. **背景介绍** | BFF/这个领域解决什么问题，痛点是什么 | 150-200字 | 给小白读者铺垫 |
| 3. **核心思路** | 项目核心设计思想，它说的零成本到底是怎么做到的 | 250-350字 | 讲清楚核心价值 |
| 4. **实战集成** | 我是怎么集成到项目里的，步骤和代码示例 | 250-350字 | 实用性，读者可以照着做 |
| 5. **踩坑记录** | 真实遇到的问题，心态崩溃的时刻 | 300-400字 | 真实，反AI，价值密度高 |
| 6. **数据说话** | 压测对比，性能数据，客观分析 | 150-250字 | 说服力，不空谈 |
| 7. **优缺点总结** | 清晰列出优缺点，适用/不适用场景 | 150-200字 | 给读者明确建议 |
| 8. **感悟升华** | 技术选型思考，个人感悟，总结 | 150-250字 | 超越项目本身，给读者启发 |

### 字数控制
- 全文: 2,000-3,000字（中文），约4,000-6,000字符
- 预估阅读时间: 8-10分钟，符合公众号阅读习惯
- 太短说不清楚，太长读者没耐心
- 数据验证: 此区间完读率最佳，当前三篇文章都符合要求 ✅

### 段落结构要求
- **所有段落控制在 3-4 行**，禁止大段文字堆砌
- 每1000字至少包含 **2 个代码块**
- 使用 emoji 和标题层级增强可读性
- 黄金 3-4 行段落让阅读压力最小化，提升完读率

## 四、内容风格要求

### ✅ 必须做到
1. **口语化表达**: 至少5处口语化用词
   - 说实话、哈哈、心态崩了、真要命、眼睛都亮了、冲昏头脑...
   - 像跟朋友聊天一样，不是写论文

2. **真实体验**: 必须是自己用过一段时间的真实感受
   - 不说谎，不吹项目，不回避缺点
   - 从期待→踩坑→务实的心路历程完整呈现

3. **反AI检测**:
   - 避免 "首先/其次/最后" 这种生硬结构
   - 允许一些不确定语气 ("我觉得"、"可能"、"我踩了这个坑你不一定会")
   - 分享失败经历，失败比全成功更真实

4. **有用**: 读完必须给读者带走点什么
   - 至少10条具体的经验/教训/建议
   - 明确告诉读者什么情况应该用，什么情况别用

5. **代码示例**: 至少2-3段关键代码配置
   - XML依赖、Java配置、DSL示例
   - 读者可以复制粘贴参考

### ❌ 避免
- 避免翻译官方文档
- 避免只吹优点不踩坑
- 避免空洞的概念堆彻
- 避免生硬的AI结构化写作
