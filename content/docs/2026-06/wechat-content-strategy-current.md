## 📐 内容规范

### 1. 字数要求
- **目标范围**: `2000 - 3000` 字
- **最近表现**: 连续3篇都在 2400-2600 之间 ✓
- **为什么**: 微信公众号读者注意力黄金长度，太长没人看，太短说不清楚

### 2. 结构要求（强制六段式）

必须按照以下顺序：
1. **背景** → 为什么做这个项目/技术，解决什么问题
2. **核心思路** → 整体架构/设计思路是什么，核心原理
3. **实战** → 实际怎么做，有代码示例
4. **踩坑** → 至少 4-5 个真实踩坑经历，每个坑给解决方案
5. **数据** → 实际数据对比，效果说话，不空谈
6. **感悟** → 个人总结，经验教训，一句话点醒读者

**为什么**: 固定结构读者容易阅读，作者不容易漏项，保证文章质量稳定。

### 3. 反AI检测要求

✅ **必须做**:
- 大量**口语化表达**（至少 5 处）：说实话、心态崩了、真没想到、忍无可忍...
- **真实失败经历**：不只说成功，说清楚踩了什么坑
- **不确定语气**：说实话、我觉得、可能、不一定...
- 结尾**开放式互动**：问读者一个问题，引导评论

❌ **禁止做**:
- 禁用 `首先/其次/最后` 这种结构化词汇
- 禁用太工整的排比
- 不要官话套话，说人话

## 🏷️ 标题优化策略（2026-06-28 更新）

### 旧策略: 总结性标题
- 优点: 信息完整，读者知道讲什么
- 缺点: 太平铺直叙，点击率不高

### 新策略: 悬念+冲突+口语化
- **钩子前置**，第一时间抓住注意力
- **用数字**，具体坑数/经验数增加可信度
- **口语化**，不用书面语

#### 对比示例:

| 旧标题 | 优化后新标题 |
|--------|-------------|
| `MCP 生产环境 Docker 部署：我踩了 94 个坑后整理的开箱即用配置` | `部署MCP踩了94个坑后，我掏出这套能直接用的Docker配置` |
| `我给狗狗做了个AI推荐散步路线的APP，三个月踩坑总结，说实话真没想到这么多问题` | `给狗狗做AI遛狗APP三个月，这几个坑真没想到` |
| `用了Capa-BFF三个月，聊聊零成本BFF到底香不香` | `用了Capa-BFF三个月，零成本BFF真的香吗？` |

### 标题公式
> `[场景/项目] + [数字/结果] + [悬念/冲突]`

## 🎣 开头Hook优化（2026-06-28 更新）

### 旧写法:
```
说实话，我一开始真没想到部署 MCP 服务器和普通 REST API 差这么多。
你按照网上教程写好 Spring Boot 应用...一推到生产，莫名其妙的问题就开始了。
```

### 优化写法:
```
你按照网上教程部署MCP，本地跑一切正常，一推到生产就各种卡？
我前后踩了94次坑，现在掏出这套直接能用的配置，复制粘贴就能用。
```

### 优化要点:
1. **第一句就命中读者痛点**
2. **直接给出收益预期**
3. **不要铺垫太长**，3句话内进入主题

## 📌 核心原则

> **真实 > 完美**
>
> 分享真实踩坑经验比吹牛逼更能建立信任，读者能真正学到东西。
