
# 用了三个月 Vibe-Memory 说说真心话：终于解决了AI记不住我的尴尬问题

说实话，我之前也被这个问题折磨很久了。

每次跟ChatGPT聊天，它都像鱼的记忆一样 —— 上次聊了一半，下次重启对话它就啥都忘了。你得重新跟它说一遍你是谁、你在做什么项目、遇到了什么问题，我有时候光是上下文就要发个两三百字，挺烦的真的。

哈哈，其实这个问题困扰我好久了，直到三个月前我自己动手写了 **Vibe-Memory** 这么一个小东西，才终于解决了这个心病。今天就跟大家掏心窝子聊聊，这三个月用下来到底是什么感受，可能对你也有启发。

## 为什么我非要自己写一个？

其实市面上已经有不少AI记忆方案了，Notion AI、ChatGPT自定义GPT、还有各种第三方知识库工具。但我用下来总觉得差点意思：

- 要么太贵，动不动就订阅制，我个人用用而已，每个月几刀累积起来也挺心疼
- 要么隐私不安全，把所有个人记忆都上传给第三方服务器，我总觉得怪怪的
- 要么太复杂，一大堆依赖要部署，搞完我都没精力用了
- 要么就是不够智能，只能关键字搜索，找不到语义相关的内容

我觉得吧，AI已经够聪明了，它缺的不是一个新大脑，缺的是一个**能随时掏出正确记忆的笔记本**。AI自己会推理，你只需要把相关的记忆片段给它就行，剩下的它自己搞定。

所以我就想做一个简单的：
- 数据都存在我自己服务器，隐私安全
- 够用就行，不搞花里胡哨的功能
- 便宜，最好几乎免费
- 代码干净，容易二次开发

## 最终做出来是什么样？

其实整个架构简单到你不敢信，我觉得可能因人而异，但是对我来说这个简单架构真的够用了。

```go
// 核心就这么几个概念，真的非常简单

type VibeMemory struct {
    db *pgxpool.Pool       // PostgreSQL + pgvector 存向量
    embedder Embedder      //  embedding 模型
}

// 添加记忆
func (m *VibeMemory) AddMemory(ctx context.Context, content string, metadata Metadata) error {
    embedding, err := m.embedder.Embed(ctx, content)
    if err != nil {
        return err
    }
    // 存进去，带时间戳和去重标记
    return m.store.Save(ctx, content, embedding, metadata)
}

// 搜索相关记忆
func (m *VibeMemory) Search(ctx context.Context, query string, limit int) ([]MemoryResult, error) {
    queryEmbedding, err := m.embedder.Embed(ctx, query)
    if err != nil {
        return nil, err
    }
    // 向量相似度搜索，按时间加权 + 去重
    return m.store.Search(ctx, queryEmbedding, limit)
}
```

真的就这么点东西！核心依赖就是：
- **Go** —— 编译快，单二进制，部署方便
- **PostgreSQL + pgvector** —— 不需要额外的向量数据库，你已有PostgreSQL就能用
- **OpenAI embedding**（或者本地模型）—— 我平时用 OpenAI 的 text-embedding-3-small，性价比真的高

## 几个我踩过的坑，分享给大家

说实话，做这个项目过程中，有几个问题真的没想到，踩完坑才明白：

### 1. 保存什么比怎么搜索更难

我之前以为核心是向量搜索，做着做着发现不对 —— **最难的是决定什么该保存，什么不该保存**。

你想啊，如果你跟AI聊什么都存，那搜索出来一堆不相关的垃圾，反而影响效果。我现在的策略是：

- 主动保存重要内容：项目想法、遇到的问题、解决方案、结论
- 不重要的闲聊就不存 —— 省空间，搜索也干净
- 定期清理，删掉没用的内容

这个悟点开窍之后，整个项目的实用性瞬间上去了。

### 2. Embedding 质量真的没你想的那么重要

我刚开始还纠结，是不是要用最好最大的模型？试了一圈下来发现：

**text-embedding-3-small 512dim 真的就够用了**，个人使用场景完全没问题。降维到512只损失2%相关性，但省了三分之二存储空间，性价比拉满。

如果想本地部署，`all-MiniLM-L6-v2` 也够用，81%相关性，完全免费，适合隐私敏感的朋友。我测下来`nomic-embed-text-v1.5` 确实更好一点，但体积也大一点，看你取舍。

有一点我确定：**不要用通用大模型做embedding** —— 慢，贵，质量还不如专门的小模型，完全没必要。

### 3. 隐私真的会改变架构

因为所有数据都存在我自己的数据库里，反而让架构变得简单了。你不需要搞复杂的同步、不需要搞云端处理，API只需要做两件事：添加记忆和搜索记忆。

而且有意思的是，AI每次只拿到当前查询相关的记忆片段，你不需要把整个记忆库都传给它，既省token又安全。

### 4. 简单比复杂好

我刚开始还想加缓存、加预处理、加各种高级排序... 最后发现：**Go + PostgreSQL + pgvector 真的就足够用了**。

对于个人使用，一万条记忆以内，这个组合响应都在100ms以内，完全感受不到延迟。真的没必要上什么专业向量数据库，大材小用。

## 实际用起来效果如何？

说点真实感受：

**优点：**
- ✅ 真的解决问题！现在AI终于认识我了，知道我在做什么项目，之前聊过什么
- ✅ 便宜到惊人 —— 三个月下来总共花了$2.17，真·一杯咖啡钱用一年
- ✅ 隐私完全可控 —— 所有数据都在你自己这儿，不上传全部记忆给OpenAI
- ✅ 部署简单 —— 懂点Go和PostgreSQL就能跑起来
- ✅ 代码干净 —— 整个核心没多少行，想改啥自己改

**缺点：**
- ❌ 需要一点技术能力部署，如果你完全不会敲代码，那用起来还是有点麻烦
- ❌  embedding 不是 100% 准确，偶尔会搜出来不相关的，但是概率不高
- ❌ 目前不支持多人协作，就是给个人用的
- ❌ 网页UI比较简单，我主要是通过API调用，所以没花精力做很炫的界面

## 谁适合用这个项目？

如果你符合下面几条，我觉得你可以试试：

- 👤 个人开发者，经常跟AI聊天，烦了它每次都失忆
- 🔒 在乎隐私，不想把所有个人想法都上传给第三方
- 💰 想省钱，不想每个月交订阅费
- 🛠️ 会一点点基本的部署，能搞定PostgreSQL就行

如果你是团队协作、或者需要很炫的界面，那可能这个项目不适合你。我觉得吧，工具就是这样，没有银弹，适合自己最重要。
