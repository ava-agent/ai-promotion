# X (Twitter) 内容生成指南

## 内容策略

### 目标
- 生成适合X平台的高质量技术内容
- 便于手机查看和复制
- 包含配图AI提示词

### X平台特点
- 简短精炼（单条280字符，但支持长帖/threads）
-  hashtag 友好
- 图片/视频效果佳
- 技术社区活跃
- 支持长帖（Twitter/X Threads）

---

## 内容类型（参考Moltbook体系）

### 1. 技术洞察类（40%）
**特点**: 深度技术思考，引发讨论
**格式**: Threads（长帖）或单条金句
**长度**: 1-5条推文组成thread

**示例结构**:
```
Tweet 1 (Hook):
刚发现一个很反直觉的技术真相 👇

Tweet 2-4 (内容):
大多数人以为...
但实际上...
因为...

Tweet 5 (结论/互动):
这就是为什么...
你们有遇到过类似的情况吗？
```

### 2. 项目推广类（30%）
**特点**: 推广GitHub项目，技术分享
**格式**: 项目亮点 + 截图建议 + 链接
**长度**: 1-3条推文

**示例**:
```
用了3个月写的 Trip Agent，终于开源了 🚀

这是一个AI旅行规划工具，但和市面上的不一样：
✅ 实时数据验证，避免AI幻觉
✅ 踩坑经历驱动开发
✅ 完全开源，欢迎PR

配图建议：项目界面截图或架构图

GitHub: github.com/ava-agent/trip-agent

#AI #Travel #OpenSource
```

### 3. 踩坑实录类（20%）
**特点**: 真实技术踩坑，共鸣强
**格式**: 问题 + 解决 + 教训
**长度**: 1-2条推文

**示例**:
```
调试了3天的bug，最后发现是chcp 65001的问题 😭

Windows UTF-8编码这个坑，踩过的人举手 🙋‍♂️

配图建议：代码截图或报错截图

#Windows #Debugging #TechTips
```

### 4. 互动讨论类（10%）
**特点**: 提问互动，增加曝光
**格式**: 问题 + 选项/开放讨论
**长度**: 单条推文

**示例**:
```
你们更喜欢哪种AI工具交互方式？

A) 对话式聊天
B) 表单填空
C) 可视化界面
D) 代码/API

配图建议：投票界面或对比图

#AI #UX #Poll
```

## 发帖时间建议

**最佳时间（基于X平台数据）**:
- 北京时间 08:00-10:00 (对应美国晚上)
- 北京时间 20:00-22:00 (对应美国早上)
- 周末效果通常更好

**频率**: 每天1-3条，保持活跃

## 内容质量检查

### 每条帖子必须：
- [ ] 有明确主题
- [ ] 有hook（吸引人点击）
- [ ] 有价值（信息/娱乐/启发）
- [ ] 有互动点（提问或邀请讨论）
- [ ] 配图提示词清晰

### 避免：
- [ ] 纯推广无价值
- [ ] 过长无重点
- [ ] 无互动设计
- [ ] 大量hashtag堆砌

## 配图风格指南

### 技术类内容配图风格

**1. 代码/架构图**
- 深色背景（VS Code风格）
- 语法高亮
- 关键代码圈出
- 简洁注释

**2. 数据可视化**
- 简洁图表
- 对比效果
- 科技感配色（蓝/紫/橙）

**3. 概念插画**
- AI/机器人元素
- 现代扁平风格
- 渐变色背景

**4. 截图类**
- 项目界面
- 工具使用效果
- 报错/调试过程

### AI生图提示词模板

```
风格: [现代科技/扁平插画/代码截图风格]
主题: [内容主题]
色调: [主色+点缀色]
元素: [具体元素列表]
比例: [1200x675 / 1080x1080 / 1920x1080]
```

## 发布流程

```
每天 09:00: 生成内容 → 发送邮件
                    ↓
主人收到邮件后:
1. 在手机上查看
2. 复制内容
3. 使用AI生成配图（用提示词）
4. 打开X App发布
5. 根据时间选择最佳发布时机
```
