## 📝 各平台风格对比

### 1. 小红书 (xiaohongshu)

**核心特点**: 生活化、情绪化、去AI化

**语气**: 像朋友聊天，但禁用"姐妹们"等过度亲密词
- ✅ "说实话..."
- ✅ "最近在做..."
- ❌ "姐妹们！"
- ❌ "救命！"

**长度**: 300-800字

**结构**: 
- 情绪化开场
- 1️⃣2️⃣3️⃣ 分点
- ✅❌⚠️ 优缺点
- 互动结尾

**Emoji**: 5-8个，点缀而非装饰

**标签**: 3-5个 #话题

---

### 2. 微信公众号 (wechat)

**核心特点**: 深度技术文章、实战导向

**语气**: 有经验的前辈分享踩坑心得
- 专业但不枯燥
- 深入浅出 > 炫技
- 实战 > 理论

**长度**: 1500-3000字（推荐）

**结构**:
```
## 标题
**导语**（2-3行，说清本文解决什么问题）
---
### 01. 背景与问题
### 02. 核心思路 / 技术选型
### 03. 实战落地
### 04. 踩坑与优化
### 05. 效果与数据
### 06. 写在最后
---
**推荐阅读**
```

**排版**:
- 每段3-5行（手机一屏）
- **加粗** 突出关键结论
- `> 引用块` 标注金句
- `---` 分隔线制造呼吸感
- 代码块完整可运行

**图片**: 3-8张，架构图、代码截图、效果截图

---

### 3. X/Twitter (x)

**核心特点**: 简洁观点、技术社区对话

**语气**: 直接、自信、有主见
- 像跟同行聊天
- 简洁 > 详尽
- 观点 > 描述

**长度**: 
- 单条：280字符内
- Thread：3-10条

**结构**:
```
1/ [钩子：核心结论或反直觉发现]
2/ 背景：[简短说明]
3/ 问题/方案：[具体描述]
4/ 数据：[具体数字]
5/ 总结 + 互动 + #标签
```

**风格**:
- 短句为主，删掉修饰词
- 有观点，敢于下判断
- 中英混排自然
- 数据说话（star数、性能数据）

**标签**: 最多2-3个 #buildinpublic #AI #opensource

**Emoji**: 每条0-2个，不要过多

## 🔧 MCP调用方式

### 获取平台风格模板
```bash
# 小红书
mcporter call content-styles.get_platform_prompt platform=xiaohongshu content_type=note mode=rewrite

# 微信公众号
mcporter call content-styles.get_platform_prompt platform=wechat content_type=article mode=rewrite

# X/Twitter
mcporter call content-styles.get_platform_prompt platform=x content_type=post mode=rewrite
```

### 转换内容风格
```bash
mcporter call content-styles.convert_content markdown_content="# 原始内容" platform=xiaohongshu mode=rewrite
```
