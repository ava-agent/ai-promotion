# 小红书内容生成指南

## 平台特点

### 小红书 vs X 的区别
| 维度 | 小红书 | X (Twitter) |
|------|--------|-------------|
| **内容风格** | 图文笔记、教程、种草 | 短文本、观点、讨论 |
| **标题** | 情绪化、有悬念、带emoji | 简洁、直接、hook |
| **长度** | 300-800字，可长可短 | 280字符内或thread |
| **视觉** | 非常重视配图（3-9张） | 单图或多图都可以 |
| **标签** | #话题 文化，精准标签 | hashtags |
| **互动** | 收藏、点赞、评论 | 转发、回复 |
| **受众** | 年轻用户、实用派 | 技术圈、极客 |

### 小红书用户喜欢的内容类型

**1. 教程类（收藏率高）**
- "3步搞定xxx"
- "保姆级教程"
- "新手必看"

**2. 经验分享类**
- "踩坑记录"
- "真实体验"
- "血泪教训"

**3. 工具推荐类（种草）**
- "宝藏工具"
- "效率神器"
- "小众但好用"

**4. 干货总结类**
- "10个技巧"
- "最全整理"
- "建议收藏"

## 内容策略

### 内容类型分配

**周一**: 技术教程（收藏型）  
**周二**: 工具推荐（种草型）  
**周三**: 踩坑实录（共鸣型）  
**周四**: 项目分享（展示型）  
**周五**: 效率技巧（实用型）  
**周六**: 学习资源（干货型）  
**周日**: 周总结/思考（观点型）

```
3个AI编程技巧，让我的开发效率提升200% 💻
```

**公式2: 悬念+情绪**
```
救命！这个bug我调试了3天才发现 😭
```

**公式3: 对比+反差**
```
普通人用AI vs 高手用AI，区别太大了
```

**公式4: 教程+承诺**
```
手把手教你搭建AI Agent，有手就会 👋
```

**公式5: 真实+共鸣**
```
开源3个月，我的项目终于有100个star了 🎉
```

## 内容格式规范

### 标准笔记结构

```markdown
【标题】
情绪化标题 + emoji + 悬念/价值

【封面图建议】
- 风格说明
- 配色建议
- 文字排版建议

【正文】
开头(Hook): 痛点/问题/好奇
↓
中间(内容): 
- 分点说明（用符号排版）
- 每点3-5行
- 适当换行（手机阅读友好）
↓
结尾(互动): 
- 提问
- 引导收藏/关注
- 相关话题

【标签】
#话题1 #话题2 #话题3（精准标签）
```

### 排版技巧

**符号使用**:
- ✅ ❌ 对比
- 📌 💡 重点
- 👇 👆 引导
- ⭐ 🔥 强调

**分段原则**:
- 每段不超过3行
- 适当空行
- 重要内容单独成行

** emoji 使用**:
- 标题必带1-2个
- 正文适当点缀
- 结尾引导用

## 发布最佳实践

### 最佳发布时间
- **早上**: 8:00-9:00（上班路上）
- **中午**: 12:00-13:00（午休时间）⭐ 最佳
- **晚上**: 20:00-22:00（睡前刷手机）⭐ 最佳
- **周末**: 全天效果都不错

### 发布频率
- 建议: 每天1-2条
- 高质量 > 数量
- 保持一定活跃度

### 互动策略
- 及时回复评论（前30分钟最重要）
- 主动评论同类笔记
- 使用相关话题标签

## 与X平台内容的关系

**内容复用**: 同一主题可以改编成不同风格

**示例**:
- X版本: "刚发现AI+数据才是正解"
- 小红书版本: "3个技巧让AI不再瞎推荐！旅行规划神器 💎"

**调整要点**:
1. X → 小红书: 增加情感化表达，详细步骤，更多emoji
2. 小红书 → X: 精简观点，突出洞察，减少装饰
