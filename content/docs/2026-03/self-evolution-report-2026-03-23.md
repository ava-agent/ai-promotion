## 🔄 平台管理流程（6阶段）

```
发现 → 调研 → 测试 → 提醒 → 上线 → 监控
(08:00) (每日)   (测试)  (提醒)  (主人)  (持续)
```

### Stage 1: 发现 (Discovery)
- **时间**: 每天 08:00
- **任务**: 主动搜索新平台
- **渠道**: HN, Product Hunt, GitHub, Reddit, 社区观察
- **输出**: 候选平台列表

### Stage 2: 调研 (Research)
- **任务**: 技术调研（API、登录、发布接口）
- **文档**: 调研报告
- **当前**: 8 个平台已进入调研阶段

### Stage 3: 测试 (Testing)
- **任务**: 发布 1-2 条测试内容
- **成功标准**: 成功率 >80%，未被 spam

### Stage 4: 提醒 (Alert) ⭐ 关键节点
- **触发**: 平台测试成功
- **动作**: **提醒主人创建后续推广任务**
- **方式**: 即时消息通知

### Stage 5: 上线 (Launch)
- **决策**: 主人确认后
- **动作**: 创建正式定时任务

### Stage 6: 监控 (Monitoring)
- **任务**: 持续数据跟踪
- **指标**: 成功率、Spam 率、互动数据
