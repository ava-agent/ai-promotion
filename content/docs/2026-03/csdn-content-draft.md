## 文章正文

说实话，第一次用 AI 生成视频的时候，我整个人都兴奋了。输入一段文字，等个几分钟，就能拿到一段看起来还挺专业的视频？这简直是魔法好吧。

然后我就兴冲冲地开始折腾，结果——说实话，踩的坑比我想象的多得多。

## 起因：想做个"一句话生成视频"的工具

项目背景是这样的：我们想做一个 MCP 服务，让用户可以通过自然语言描述，直接生成视频。听起来很简单对吧？就是一个 prompt → video 的过程。

技术栈选得也挺"时髦"：
- **MCP 协议**做服务通信
- **FFmpeg** 做视频处理
- **Python** 写核心逻辑
- **各类 AI API** 生成素材

信心满满地开干。

## 踩坑实录

### 坑 1：AI 生成的图片，根本不是视频的好素材

刚开始我的想法很天真：让 AI 生成几张图，然后用 FFmpeg 串成视频，配个音不就完事了？

结果发现——**静态图片做成视频，效果贼差**。

问题在哪？
- AI 生成的图虽然好看，但是**完全没有连贯性**
- 第一帧是"一个机器人在实验室"，第二帧变成"机器人在海边"，第三帧又成了"机器人在太空"...
- 这做个屁的视频啊，这是 PPT 好不好

**解决思路**：
后来只能改用专门的视频生成模型（比如 Runway、Pika 这些），而不是用图片生成+拼接的方式。成本上去了，但效果至少能看了。

### 坑 2：FFmpeg 的参数，调到我怀疑人生

FFmpeg 这玩意儿，怎么说呢... 功能确实强大，但文档写的是给人看的吗？

我想实现一个简单的功能：把生成的视频片段拼接起来，加个背景音乐，再压个码率。

然后我就在 Stack Overflow 上泡了整整两天。

**崩溃时刻**：
```
ffmpeg -i input1.mp4 -i input2.mp4 -filter_complex "[0:v][1:v]concat=n=2:v=1:a=0[outv]" -map "[outv]" output.mp4
```

看着这段命令，你知道我当时什么心情吗？我明明是学计算机的，为什么感觉自己像个文盲？

而且最烦的是，不同版本的 FFmpeg，同样的命令效果还不一样。我本地跑得好好的，部署到服务器上就不行了，然后发现服务器版本是三年前的...

**血泪教训**：
- 一定要用 conda 或者 docker 固定 FFmpeg 版本
- 复杂的 filter_complex 先用小文件测试
- 编码参数不要瞎改，H.264 虽然老但是兼容性最好

### 坑 3：视频生成 API 的"异步地狱"

现在的 AI 视频生成 API，基本都是异步的：你提交个任务，它告诉你"稍等哈，大概 2-5 分钟"，然后你自己去轮询结果。

听起来合理对吧？但实际操作起来...

**问题 1：超时处理**
有些任务生成到一半就卡住了，API 也不报错，就是永远"processing"。你不设置个超时机制，这任务能挂到天荒地老。

**问题 2：并发限制**
大部分 API 都有并发限制，你想同时生成 10 个视频？不好意思，先排队吧。那我们就要自己实现一个队列系统。

**问题 3：失败重试**
网络抖一下，或者 API 抽个风，任务就失败了。失败之后要不要重试？重试几次？重试间隔多久？这些问题看着简单，真写起来全是细节。

我们最后写了个简单的任务队列，用 Redis 存状态，大概长这样：

### 坑 4：文件存储和路径，永远的痛

开发的时候，我本地用的是 Windows，路径是 `C:\Users\...` 这种。部署到 Linux 服务器上，路径变成 `/home/...`。

然后 FFmpeg 命令里硬编码了一些路径，上线之后各种报错找不到文件。

**更坑的是**：有些视频生成 API 返回的是临时 URL，有效期只有几个小时。你要是忘了及时下载，链接失效了，任务就白跑了。

### 坑 5：成本控制，真他喵的贵

这个可能不算技术坑，但确实是很大的坑。

AI 视频生成，是真的贵。一开始我没太在意，想着"反正先跑起来再说"。结果测试了几天，账单出来吓我一跳——够我喝一个月奶茶了。

后来不得不加了一堆限制：
- 每个用户每天最多生成 3 个视频
- 单个视频最长 10 秒
- 分辨率限制在 720p

## 现在的状态

折腾了大概两周，现在终于能稳定运行了。

- ✅ 支持通过 MCP 协议调用
- ✅ 实现了任务队列和失败重试
- ✅ 成本控制还算合理
- ⚠️ 视频质量还有提升空间
- ❌ 生成速度还是慢（这个是 API 限制，没办法）

如果你也想做类似的东西，我的建议是：**先想清楚成本和场景，别像我一开始那样盲目乐观**。

## 总结

说实话，做这个项目之前，我以为"AI 视频生成"就是个调用 API 的简单活儿。真正做下来才发现，要把体验做好，需要考虑的东西太多了：错误处理、超时控制、并发管理、成本控制...

这也是我这半年折腾 AI 工具最大的感受：**Demo 容易做，产品难打磨**。
