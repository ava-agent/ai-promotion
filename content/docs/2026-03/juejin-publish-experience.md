## 🚀 发布方式

### 当前方案：API 直接调用 ✅
掘金使用 `juejin_client.py` 通过 **HTTP API** 直接发布文章，无需浏览器自动化。

**核心文件**:
- `skills/juejin-publisher/juejin_client.py` - API 客户端
- 配置方式: Cookie 认证

**发布流程**:
```
1. create_draft() - 创建草稿
2. update_draft() - 更新草稿（可选）
3. publish_article() - 发布文章
```

## ⚠️ 关键修复记录

### 修复 1: 标签单一问题 ⭐ 2026-03-30

**问题**: 所有文章只用"前端"标签，AI项目无法触达AI兴趣用户

**修复方案**:
```yaml
# AI项目标准标签组合
标签:
  - AI (必须)
  - 人工智能 (必须)
  - 开源 (推荐)
  - Python/前端/后端 (根据技术栈)

分类: 人工智能 (AI项目必须使用！)
```

**验证**: 2026-03-30 更新的内容策略中已强制执行

---

### 修复 2: 互动率为0问题 ⭐ 2026-03-30

**问题**: 所有文章评论数为0，提问式结尾效果差

**修复方案**:
```markdown
# 旧方式（失效）❌
"你们平时用什么工具？"

# 新方式（有效）✅
1. 求助式: "有个问题困扰我很久了：XXX，有大佬知道怎么解决吗？"
2. 投票式: "你们更倾向哪种方案？A.XX B.XX C.XX"
3. 争议式: "说实话，我觉得XXX其实挺鸡肋的，你们觉得呢？"
```

**验证**: 已纳入内容策略，下次发文必须使用新互动模板

---

### 修复 3: 标题同质化问题 ⭐ 2026-03-30

**问题**: "说实话"/"老实说"使用频率过高，读者疲劳

**修复方案**:
```markdown
# 标题风格轮换（优先级排序）
1. 【带源码】实战型: 30%
2. 口语化+自嘲: 30%
3. 提问型: 20%
4. 踩坑型: 10%
5. 数字型: 10%

# 轮换规则
- 同一开头词连续使用不超过1次
- 每次标题使用不同模板
```

**验证**: 已建立标题模板库，A/B测试中

---

### 修复 4: 发布时间偏晚 ⭐ 2026-03-30

**问题**: 多在23:00后发布，错过流量高峰

**修复方案**:
```yaml
最佳发布时间:
  16:00: ⭐⭐⭐⭐⭐ 最高优先级（下午摸鱼高峰）
  12:00: ⭐⭐⭐⭐ 高优先级（午休时间）
  20:00: ⭐⭐⭐ 中优先级（晚上空闲）
  
避免:
  - 23:00+ （流量低）
  - 周末早晨
```

**验证**: 已更新发布策略，优先16:00发布

## 🔧 技术实现细节

### API 端点
```python
BASE_URL = "https://api.juejin.cn"

# 创建草稿
POST /content_api/v1/article_draft/create

# 发布文章
POST /content_api/v1/article/publish
```

### 关键参数
```python
{
    "title": "标题（1-100字符）",
    "content": "Markdown内容",
    "category_id": "分类ID",
    "tag_ids": ["标签ID1", "标签ID2"],  # 最多5个
    "is_original": 1,  # 1=原创, 0=转载
    "edit_type": 10,   # 10=Markdown
    "brief_content": "简介（自动截取前100字）"
}
```

### Cookie 管理
- Cookie 存储在 `skills/juejin-publisher/config.yaml` 或 `.env`
- 需要定期更新（建议每月检查一次）
- 更新方式: 登录掘金网页 → F12 → Application → Cookies → 复制完整Cookie

## 📝 发布前检查清单

### 技术检查
- [ ] Cookie 未过期
- [ ] API 调用返回 200
- [ ] 草稿创建成功

### 内容检查
- [ ] 标题通过风格检查（非"说实话"模板）
- [ ] Hook 使用新类型（冲突/好奇/反转）
- [ ] 口语化表达 ≥5处
- [ ] 情感表达 ≥2处
- [ ] 真实场景 ≥1处
- [ ] 精确数字 ≤2个

### 标签检查 ⭐ 重点
- [ ] AI项目必须使用 `AI` + `人工智能` 标签
- [ ] 开源项目添加 `开源` 标签
- [ ] 分类选择 `人工智能`（AI项目）
- [ ] 标签数量 2-5个

### 互动检查 ⭐ 重点
- [ ] 结尾使用求助式/投票式/争议式
- [ ] 避免泛泛的提问
- [ ] 明确引导评论

### 时间检查
- [ ] 优先选择 16:00 发布
- [ ] 备选 12:00 或 20:00
- [ ] 避免 23:00+

## 🎯 高效果标题模板（已验证）

### 1. 【带源码】实战型 ⭐⭐⭐⭐⭐
```
【带源码】我用[技术]写了/实现了[项目]
```
示例: "【带源码】我用 AI 写了个摩旅路线规划工具"

### 2. 提问好奇型 ⭐⭐⭐⭐
```
为什么[现象]？[解释]
```
示例: "为什么摩旅App不敢推荐真正的烂路？我找到了原因"

### 3. 踩坑实录型 ⭐⭐⭐⭐
```
踩了个大坑：[具体问题]
```
示例: "踩了个大坑：GPX文件解析把我坑惨了"

### 4. 口语化+自嘲 ⭐⭐⭐⭐
```
说实话，[动作][结果/状态]
```
示例: "说实话，我做这个项目是因为自己记性太差了"

### 5. 数字极简型 ⭐⭐⭐
```
[数字][单位]，[结果/动作]
```
示例: "折腾三个月，我把摩旅路线和AI搞在一起了"
