# 平台管理流程

## 🔄 平台生命周期管理

**文档目的**: 规范新平台从发现到上线的完整流程  
**适用范围**: 所有社交媒体平台  
**责任人**: 旺财（自主执行）+ 主人（关键决策）

---

## 📋 阶段划分

### Stage 1: 发现 (Discovery)

**触发条件**: 每天 08:00 定时任务  
**执行者**: 旺财（自主探索）  
**输出**: 候选平台列表

**执行内容**:
1. 主动搜索新平台（HN, Product Hunt, GitHub, Reddit）
2. 观察社区分享的平台
3. 评估技术趋势和垂直领域

**评估标准**:
- [ ] 支持 API/自动化发布？
- [ ] 受众与主人内容匹配？
- [ ] 平台活跃度和用户质量？
- [ ] 技术实现难度可接受？

**文档更新**:
- 更新 `memory/platform-discovery-log.md`
- 添加到"候选平台池"

---

### Stage 2: 调研 (Research)

**触发条件**: 主人确认后，或高优先级平台自动进入  
**执行者**: 旺财（自主执行）  
**输出**: 调研报告

**执行内容**:
1. 检查登录状态，获取 Cookie/Token
2. 阅读 API 文档
3. 测试发布接口
4. 记录内容格式、审核时间、限制规则

**调研清单**:
- [ ] API 文档链接
- [ ] 登录方式（Cookie/Token/OAuth）
- [ ] 发布接口端点
- [ ] 内容格式（Markdown/HTML/纯文本）
- [ ] 图片支持情况
- [ ] 链接策略（允许/禁止/限制）
- [ ] 审核时间和机制
- [ ] 标签系统
- [ ] 最佳发布时间

**定时任务**:
- 每天特定时间执行（如 09:00, 09:30 等）
- 任务名称: "[平台名] Platform Research"

**文档更新**:
- 更新 `memory/new-platforms-config.md`
- 创建平台调研报告

---

### Stage 3: 测试 (Testing)

**触发条件**: 调研完成，技术可行  
**执行者**: 旺财（自主执行）  
**输出**: 测试结果

**执行内容**:
1. 发布 1-2 条测试内容
2. 观察审核时间和通过率
3. 测试不同内容类型效果
4. 收集用户反馈数据

**测试清单**:
- [ ] 技术类长文（1500+ 字）
- [ ] 短内容/动态（300-500 字）
- [ ] 带图片的内容
- [ ] 带链接的内容（如果允许）
- [ ] 不同发布时间

**成功标准**:
- 发布成功率 >80%
- 审核通过，未被标记为 spam
- 有基本互动（点赞/评论）

**失败处理**:
- 记录失败原因
- 分析是否可解决
- 决定是否放弃或调整策略

---

### Stage 4: 提醒 (Alert)

**触发条件**: 平台测试成功，发布走通  
**执行者**: 旺财（自动提醒）  
**输出**: 提醒消息

**提醒内容**:
```
🎉 主人！新平台已走通发布流程！

平台: [平台名称]
链接: [平台链接]
测试状态: ✅ 成功

建议操作:
1. 创建正式推广任务（每日/每周）
2. 制定内容策略（类型、风格、发布时间）
3. 配置 Cookie/Token 长期存储
4. 添加到主矩阵文档

需要我立即创建正式推广任务吗？
```

**提醒方式**:
- 即时消息通知
- 更新 `memory/platform-discovery-log.md` 的"已走通平台"列表
- 添加到明日计划

---

### Stage 5: 上线 (Launch)

**触发条件**: 主人确认后  
**执行者**: 旺财（创建任务）+ 主人（审核）  
**输出**: 正式推广任务

**创建内容**:
1. 正式定时任务（如每天 10:00 发布）
2. 内容策略文档
3. 质量监控规则

**任务配置**:
```json
{
  "name": "[平台名] Daily Post",
  "schedule": "0 10 * * *",
  "platform": "[平台名]",
  "content_type": "技术分享/项目推广/经验总结",
  "word_count": "1500+",
  "language": "中文/英文",
  "link_policy": "允许/禁止"
}
```

**文档更新**:
- 更新 `memory/self-media-matrix-2026-03-23.md`
- 添加到"活跃平台"列表
- 记录到推广日志

---

### Stage 6: 监控 (Monitoring)

**触发条件**: 平台上线后持续进行  
**执行者**: 旺财（自动监控）  
**输出**: 数据报告

**监控指标**:
- 发布成功率
- Spam 率
- 互动数据（点赞、评论、分享）
- 粉丝增长
- Cookie/Token 有效期

**异常处理**:
- 发布失败 >3 次：检查 API 状态
- Spam 率 >5%：调整内容策略
- Cookie 过期：提醒主人更新

**定期报告**:
- 每周数据汇总
- 每月效果评估
- 季度平台审计（决定是否继续运营）

---

## 🚨 决策节点

### 节点 1: 是否进入调研？
**触发**: 发现新平台  
**决策者**: 旺财（自动评估）  
**标准**:
- 技术可行性：中高
- 受众匹配度：高
- 平台质量：高

### 节点 2: 是否继续测试？
**触发**: 调研完成  
**决策者**: 旺财（自动评估）  
**标准**:
- API 可用：是
- 登录可行：是
- 发布接口稳定：是

### 节点 3: 是否正式上线？
**触发**: 测试成功  
**决策者**: 主人（人工确认）  
**考虑因素**:
- 测试数据是否满意？
- 是否有精力维护新平台？
- 是否符合长期战略？

### 节点 4: 是否放弃平台？
**触发**: 长期效果不佳或技术不可行  
**决策者**: 主人（人工确认）  
**标准**:
- 连续 2 周发布成功率 <50%
- Spam 率持续 >10%
- 零互动或负面反馈
