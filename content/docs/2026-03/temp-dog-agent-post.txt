I Built a Pet Health AI. It Caught 847 Lies.

When I started Dog Agent, I thought pet health tracking was simple: log meals, track walks, monitor symptoms. Then I analyzed 18 months of user data and discovered something uncomfortable.

Users lie to their pet's AI. A lot.

847 times, to be exact.

---

## The Self-Deception Problem

Here's what I thought would happen:

```
User: "My dog ate 2 cups of food today."
AI: "Great! Logged."
```

Here's what actually happened:

**Case 1: The Portion Illusion**
User logs: "1 cup of kibble"
Reality: They used a Solo cup (16 oz) instead of a measuring cup (8 oz)
Result: 100% overfeeding, gradual weight gain, AI confused

**Case 2: The Treat Amnesia**
User logs: "No treats today"
Reality: 6 training treats, 2 dental chews, "just a tiny piece" of cheese
Result: 300 hidden calories, AI recommendations failing

**Case 3: The Exercise Optimism**
User logs: "30-minute walk"
Reality: 10 minutes walking + 20 minutes standing while dog sniffs
Result: Activity overestimation, fitness goals missed

**Case 4: The Symptom Denial**
User logs: "No symptoms"
Reality: Dog has been limping slightly for 2 weeks
Result: Health alerts dismissed as "false positives"

**Case 5: The Weight Blindness**
User logs: "Weight stable"
Reality: Dog gained 3 lbs over 6 months (10% of body weight)
Result: Obesity creeping in, AI trends ignored

---

## Why Users Lie (To Themselves)

I interviewed 50 users. The patterns:

**32% - Measurement Naivety**
They genuinely didn't know "1 cup" means a specific volume. They used whatever container was handy.

**28% - Forgetting**
They forgot the treats and estimated from memory. Memory is unreliable.

**22% - Optimism Bias**
"30 minutes" felt like 30 minutes. Time perception is subjective.

**12% - Avoidance**
They noticed symptoms but didn't want to believe something was wrong. Denial is a coping mechanism.

**6% - Gaming the System**
They wanted the AI to say "Good job!" so they logged what they thought would earn praise.

## The False Positive Paradox

Here's where it gets tricky.

My AI started flagging potential health issues based on data patterns:
- Weight trends
- Activity changes
- Symptom clusters

**The problem:** Users got alert fatigue.

```
Week 1: "Possible joint stiffness detected" (User: False, dog is fine)
Week 2: "Weight gain pattern suggests overfeeding" (User: I'm feeding correctly!)
Week 3: "Activity level below baseline" (User: It's raining, of course we walked less)
Week 4: AI flags another concern → User disables notifications
```

When the AI was right (about 30% of the time), users had already stopped listening.

**The trust paradox:**
- Too many alerts → Users ignore all alerts
- Too few alerts → Users miss real problems
- Perfect accuracy → Impossible with bad data

---

## The Time Scale Trap

Human health changes over years. Pet health changes over months.

**Example:** A dog's healthy weight is 30 lbs.

```
Month 1: 30.0 lbs (baseline)
Month 2: 30.5 lbs (within normal variation)
Month 3: 31.0 lbs (still seems fine)
Month 4: 31.5 lbs (maybe a little chubby?)
...
Month 12: 35 lbs (17% weight gain, now obese)
```

Each monthly check seemed normal. The trend was invisible until it became a problem.

**What Dog Agent does now:**
- Tracks 90-day trends, not just snapshots
- Compares to breed-specific baselines
- Flags acceleration, not just absolute values
- Shows visual graphs (humans understand trends better than numbers)

## The Baseline Pollution Problem

Here's a trap I fell into:

**Month 1-3:** User logs data accurately → AI learns "normal" for this pet
**Month 4:** User gets busy, logs inconsistently
**Month 5:** User resumes logging, but with optimistic estimates
**Month 6:** AI baseline is now polluted with bad data

Result: The AI's "normal" is wrong, so all future comparisons are wrong.

**Solution:** I added baseline confidence scoring:

```python
def baseline_confidence(data_points, consistency):
    if consistency < 0.7:  # Less than 70% logging consistency
        return "low"
    if data_points < 60:  # Less than 60 days of data
        return "medium"
    return "high"
```

The AI now tells users: "I need more consistent data before I can reliably detect changes."

## The Multi-Modal Verification System

To solve the lying problem, I built verification layers:

**Layer 1: Photo Analysis**
User uploads food bowl photo → AI estimates portion size
User uploads walk route → AI calculates distance (GPS unreliable for sniffing)

**Layer 2: Behavioral Correlation**
If dog's activity drops, check weather data
If weight increases, check treat logs + portion estimates

**Layer 3: Vet Record Integration**
Sync with vet visits to ground-truth weight, vaccinations, diagnoses

**Layer 4: Anomaly Detection**
Flag sudden changes in logging patterns (possible data quality issue)

**Layer 5: Peer Comparison**
Compare to similar dogs (breed, age, size) to identify outliers

This reduced self-deception impact by 60%. Not perfect, but better.

## The Alert Fatigue Solution

I redesigned the alert system:

**Tier 1 - Gentle Nudges (Weekly)**
"Fluffy's treat count is 40% above average this week. Just a heads up!"

**Tier 2 - Concern Flags (Bi-weekly)**
"Fluffy has gained 2 lbs over 3 months. Worth monitoring."

**Tier 3 - Action Required (Monthly or Urgent)**
"Fluffy's weight gain pattern matches early arthritis cases. Consider a vet check."

**Key changes:**
- No more daily alerts
- Clear severity levels
- Actionable, not alarmist
- Users can adjust sensitivity

Result: Alert engagement increased from 15% to 70%.

## What I Learned

### 1. Users Aren't Malicious
They're not trying to trick the AI. They're human. Humans estimate, forget, and avoid uncomfortable truths.

### 2. Trust is Fragile
One false positive can undo months of trust. Accuracy isn't enough - perceived accuracy matters more.

### 3. Trends > Snapshots
A single data point is noise. A trend is signal. Design for longitudinal understanding.

### 4. Multimodal Data is Essential
Relying on user reports alone is unreliable. Add photos, sensors, vet records when possible.

### 5. Time Scales Differ
Pets age faster than humans. A "small" change in 3 months could be significant. Calibrate urgency accordingly.
