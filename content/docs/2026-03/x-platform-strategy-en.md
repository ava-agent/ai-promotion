## 📝 内容模板

### 1. 技术洞察类 (Insight)
**目标**: 展示技术深度，引发思考
**结构**:
```
[反直觉Hook]

大多数人[常规认知]。
但真相是：[核心洞察]。

[简短解释/案例]

这就是为什么[解决方案/观点]。

[互动问题]
```

**示例**:
```
AI旅行规划的真正瓶颈不是算法，是数据质量。

大多数工具只关注生成行程，
但不知道哪家店营业、哪条路在修。

我们花了6个月验证：23%的推荐失败来自过时的POI数据。

数据+AI才是正解。

你们在做AI产品时踩过哪些数据坑？
```

**检查清单**:
- [ ] 有反直觉的Hook
- [ ] 具体数据支撑（百分比/数字）
- [ ] 真实案例或经验
- [ ] 明确的结论
- [ ] 互动问题结尾

你们有被文档坑过的经历吗？
```

**目标**: 展示系统设计能力
**结构**:

2. [决策2 + 原因]

[权衡分析]

[效果数据]

[链接/资源]
```

**示例**:
```
如何设计一个支持10万QPS的多租户系统？

目标：每个租户数据隔离，但共享基础设施。

关键决策：
1. 行级隔离 vs 分库分表 → 选择行级，减少运维复杂度
2. 同步vs异步计费 → 选择异步，保证核心链路性能

权衡：放弃了实时计费，换取了99.99%可用性。

效果：单集群支撑10万QPS，平均延迟<50ms。

详细设计：github.com/xxx/architecture-guide
```

## 🎨 风格指南

### Hook写作技巧
1. **数字冲击**: "94%的AI项目失败..."
2. **反直觉**: "最好的代码是没有代码"
3. **情感共鸣**: "花了2年才明白..."
4. **挑战权威**: "传统做法错了..."

### 语言特点
- **简洁有力**: 每句话都要有信息密度
- **主动语态**: "我们发现"而非"被发现"
- **第一人称**: 真实、个人化
- **emoji点缀**: 🚀💡😭👇（适度使用）

### 避免
- ❌ 营销语言（"革命性"、"颠覆性"）
- ❌ 过度承诺（"100%"、"完美"）
- ❌ 空洞赞美（"太棒了"、" amazing"）
- ❌ 长段落（X平台适合短内容）
