## 1. 平台概述

### 1.1 什么是 Mastodon

Mastodon 是一个**去中心化的开源社交媒体平台**，采用联邦宇宙（Fediverse）架构：

- **去中心化**: 不是单一公司控制，由多个独立服务器（实例）组成
- **开源**: 源代码公开，任何人可以搭建自己的实例
- **联邦制**: 不同实例之间可以互相通信，形成联邦宇宙
- **非营利**: mastodon.social 由 Mastodon GmbH 运营，通过 Patreon 众筹

### 1.2 核心数据 (mastodon.social)

| 指标 | 数值 |
|------|------|
| 活跃用户 | 28.6 万+ |
| 服务器位置 | 德国柏林 |
| 运营方 | Mastodon GmbH |
| 软件版本 | v4.6.0-alpha.6 |

### 1.3 内容形式

- **嘟文 (Toot)**: 类似 tweet，最多 500 字符
- **转嘟 (Boost)**: 类似 retweet
- **喜欢 (Favourite)**: 类似 like
- **话题标签 (Hashtag)**: #标签
- **媒体附件**: 图片、视频、GIF
- **投票**: 可创建投票
- **引用嘟文**: 支持引用回复

## 2. API 能力分析

### 2.1 REST API 覆盖度

Mastodon 提供非常完善的 REST API：

| 功能类别 | API 端点 | 支持度 |
|----------|----------|--------|
| **账号管理** | `/api/v1/accounts/*` | ✅ 完整 |
| **发布嘟文** | `POST /api/v1/statuses` | ✅ 完整 |
| **媒体上传** | `POST /api/v1/media` | ✅ 完整 |
| **时间线** | `/api/v1/timelines/*` | ✅ 完整 |
| **搜索** | `/api/v2/search` | ✅ 完整 |
| **通知** | `/api/v1/notifications` | ✅ 完整 |
| **关注/取关** | `/api/v1/accounts/{id}/follow` | ✅ 完整 |
| **点赞/转嘟** | `/api/v1/statuses/{id}/favourite` | ✅ 完整 |
| **预定发布** | `/api/v1/scheduled_statuses` | ✅ 完整 |
| **投票** | `/api/v1/polls` | ✅ 完整 |

### 2.2 认证方式

- **OAuth 2.0**: 标准 OAuth 流程
- **Access Token**: 应用注册后获取
- **Scope 权限**: 细粒度权限控制

### 2.3 Rate Limits（速率限制）

| 限制类型 | 限制值 | 说明 |
|----------|--------|------|
| **普通接口** | 300 次 / 5 分钟 | 每个账号 |
| **媒体上传** | 30 次 / 30 分钟 | 防止滥用 |
| **删除嘟文** | 30 次 / 30 分钟 | 防止批量删除 |
| **IP 限制** | 300 次 / 5 分钟 | 每个 IP |
| **注册账号** | 5 次 / 30 分钟 | 每个 IP |

**评价**: 限制相对宽松，适合自动化运营。

## 3. 使用条款分析

### 3.1 mastodon.social 站点规则

**允许的内容**:
- ✅ 技术讨论和分享
- ✅ 个人项目和作品推广
- ✅ 人工智能相关内容（需披露）
- ✅ 链接分享

**禁止的内容**:
- ❌ 色情或暴力内容（必须标记敏感）
- ❌ 种族主义、性别歧视、仇恨言论
- ❌ 煽动暴力
- ❌ 骚扰、人肉搜索
- ❌ 虚假信息
- ❌ **仅发布 AI 生成内容的账号**（规则明确禁止）

**关键规则解读**:

> "账户不得仅发布人工智能生成的内容"

⚠️ **重要警告**: 这意味着纯自动化的 AI 内容农场账号可能会被限制或封禁。

### 3.2 API 使用指南

官方文档明确鼓励开发应用：
- ✅ 支持第三方客户端
- ✅ 支持机器人账号（bot flag）
- ✅ 支持自动化工具
- ✅ 提供详细的开发者文档

## 6. 实施建议

### 6.1 如果决定实施

**推荐策略**:
1. **人机结合**: 不是纯自动化，而是辅助工具
2. **账号标记**: 注册时标记为 bot 账号
3. **内容混合**: AI 生成 + 人工编辑 + 真实互动
4. **频率控制**: 每天 2-3 条，避免过度刷屏
5. **标签使用**: 善用 #标签 增加曝光

**技术实现**:
```python
from mastodon import Mastodon

# 初始化
mastodon = Mastodon(
    access_token = 'your_access_token',
    api_base_url = 'https://mastodon.social'
)

# 发布嘟文
mastodon.toot('Hello, Mastodon! #opensource #AI')
```

### 6.2 内容策略

**适合的内容**:
- 技术博客链接 + 摘要
- 开源项目更新
- 技术思考/教程
- 社区互动（回复、点赞）

**避免的内容**:
- 纯营销文案
- 重复性推广信息
- 未经编辑的 AI 生成内容
- 过度自推

### 6.3 推荐优先级

**建议**: ⭐⭐⭐ 可以实施，但不是最高优先级

原因:
1. ✅ 技术社区活跃，适合技术内容
2. ✅ API 友好，开发成本低
3. ⚠️ "仅 AI 内容"禁令限制了纯自动化
4. ⚠️ 去中心化导致传播效果有限
5. ⚠️ 中文用户相对较少

**推荐顺序**:
1. Moltbook（核心，已运行）
2. 知乎（中文社区，已运行）
3. Dev.to（国际开发者，已运行）
4. 掘金（中文开发者，已运行）
5. CSDN（中文技术，已运行）
6. **Mastodon**（技术社区，可选）
7. Medium（待测试）
