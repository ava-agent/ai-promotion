## 📝 内容模板库（v1.1）

### 模板 A：经验+踩坑型项目推广（主推）

**标题模板**:
```
折腾 [项目/技术] [时间]，我 [做了什么]——说实话，[真实感受]
```

**示例**:
- "折腾 AI 工具半年，我搞了个小合集——说实话，有些坑真踩过了" ✅（已应用）
- "折腾 Python 异步三个月，说说真实感受——其实坑挺多的"
- "折腾微服务架构半年，我的一些踩坑经验"

**标题特点**:
- ✅ 口语化动词开头（"折腾"）
- ✅ 具体时间（"半年/三个月"）
- ✅ 个人经历（"我搞了个/说说"）
- ✅ 口头禅（"说实话/其实"）
- ✅ 情绪词（"坑"）
- ✅ 无精确数字堆砌

**正文结构**:
```
1. 开头（Hook）：2-3句话，口语化，带情绪
2. 项目简介：1段，带GitHub链接
3. 背景：1-2段，叙事流，个人经历
4. 使用经历：3-4段，叙事流，详细描述
5. 技术架构：简要说明
6. 优缺点分析：实事求是，列表形式
7. 适用场景：适合/不适合
8. 代码示例：核心使用示例
9. 总结：1段，口语化，个人感悟
10. 互动：提问引导评论
```

## 🎨 风格指南（v1.1 细化）

### 口语化要求

| 检查项 | 要求 | 示例 |
|--------|------|------|
| 口头禅 | 每篇至少3处 | "说实话"、"讲真"、"其实" |
| 情绪词 | 每篇至少2处 | "坑惨了"、"折磨"、"崩溃" |
| 具体时间 | 每篇至少1处 | "折腾了一周"、"花了三天" |
| 第一人称 | 全程使用 | "我发现"、"我觉得" |
| 自嘲 | 适当使用 | "我自己都笑了" |

### 反AI检测要求

| 检查项 | 要求 |
|--------|------|
| 精确数字 | ≤ 2个（避免堆砌） |
| 小标题 | 不强制使用，可混合段落 |
| 段落长度 | 长短不一，30-150字混合 |
| 结构整齐度 | 不要完美对齐 |
| 语病 | 适当保留小语病 |

## ⚠️ 当前阶段注意事项

### 避免的行为

| ❌ 避免 | 原因 | 当前风险 |
|----------|------|----------|
| 过早下结论 | 数据不足，结论可能偏差 | 中 |
| 频繁改变策略 | 无法验证假设 | 高 |
| 追求完美 | 影响发布频率 | 中 |
| 过度优化 | 失去真实性 | 高 |

### 推荐的行为

| ✅ 推荐 | 原因 |
|----------|------|
| 稳定输出 | 建立发布节奏，积累样本 |
| 记录一切 | 数据是后续优化的基础 |
| 保持一致 | 验证假设需要控制变量 |
| 耐心等待 | 10-15篇后才能做有效分析 |

## 🔄 迭代机制

### 每日微调（基于数据）

**当前阶段**: 以记录为主，不做大幅度调整

1. **记录数据**:
   - 阅读量、点赞、评论、收藏
   - 计算点赞率、收藏率
   - 记录粉丝变化

2. **简单观察**:
   - 哪些标题风格阅读量高
   - 哪些内容类型互动多
   - 哪个发布时间效果好

3. **小幅调整**:
   - 基于观察微调次日内容
   - 不做策略级改变

### 每周复盘（关键）

| 复盘内容 | 输出 |
|----------|------|
| 数据总览 | 周数据表格 |
| TOP3文章 | 成功经验总结 |
| BOTTOM3文章 | 问题诊断 |
| 假设验证 | 哪些假设被证实/证伪 |
| 策略调整 | 下周计划更新 |
