# X Developer Agreement - Use Case Description

## 申请用例描述模板

### 英文版本（推荐）

```
I am a software architect and open-source contributor developing AI tools and cloud-native infrastructure projects. I plan to use X's API for the following purposes:

1. Content Sharing:
   - Share updates about my open-source projects hosted on GitHub (github.com/kevinten10, github.com/open-octopus)
   - Post technical insights, development experiences, and architectural decisions
   - Share tutorials and best practices about multi-runtime architecture and AI agent development

2. Community Engagement:
   - Automatically post project milestones, new features, and release announcements
   - Share technical blog posts and articles from my Dev.to and Moltbook profiles
   - Engage with the developer community through technical discussions

3. Data Usage:
   - I will only access public data from my own account (@kevinten1024)
   - No access to other users' private data or direct messages
   - No automated following/unfollowing or mass interactions
   - No collection of user data for analysis or marketing

4. Technical Implementation:
   - Language: Python with X API v2
   - Authentication: OAuth 2.0 Bearer Token
   - Rate Limits: Will strictly adhere to Free tier limits (100 posts/month, 500 reads/month)
   - Automation: Scheduled posting during optimal engagement hours (morning/evening UTC)

5. Compliance:
   - All content will be original technical content created by me
   - No spam, no repetitive posting, no violation of X's Terms of Service
   - Will include attribution and proper context for all shared links
   - Will respect community guidelines and developer best practices

Projects to be promoted:
- OpenOctopus: Realm-native life agent system (github.com/open-octopus/openoctopus)
- Capa-Java: Multi-runtime SDK for hybrid cloud
- Various AI tools and developer utilities

The primary goal is to build a technical brand, share knowledge with the global developer community, and drive adoption of open-source projects.
```

## 填写要点

### ✅ 必须包含
1. **具体身份**: 软件架构师、开源贡献者
2. **具体项目**: 提及 GitHub 项目和 X 账号
3. **具体用途**: 分享项目更新、技术见解
4. **数据限制**: 只访问自己的数据，不访问他人数据
5. **合规承诺**: 遵守规则、不发垃圾信息
6. **技术细节**: 使用 X API v2、Python 等

### ❌ 避免提及
- 自动化点赞/关注
- 数据分析/挖掘
- 营销/广告目的
- 访问其他用户数据
- 大规模自动化操作
