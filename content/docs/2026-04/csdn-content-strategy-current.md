## 🎯 策略概述

### 核心目标
**优先级排序**（基于当前状态调整）:
1. 🔴 **修复发布通道** — 11 天中断，必须立即恢复

3. 🟢 **恢复数据积累** — 重新建立每日发布节奏，补齐样本

### 当前阶段重点
当前阶段从"冷启动"调整为 **"爆发式补发 + 节奏重建"**:
- **补发期**（1-2 天）: 每天发布 2 篇，快速清空 pending
- **稳定期**（第 3 天起）: 恢复每日 1 篇的正常节奏
- **数据追赶**: 2 周内积累至少 10 篇有效样本，进入真正的优化期

## 📝 内容模板库（v1.2 维持）

### 模板 A：经验+踩坑型项目推广（主推）

**标题模板**:
```
折腾 [项目/技术] [时间]，我 [做了什么]——说实话，[真实感受]
用了 [技术] [时间]，说说真实感受：[核心问题]
拿了 [成绩] 的 [技术方案]，[时间] 后我还在用
```

**近期成功示例**:
- "用了 Capa-BFF 三年，说说真实感受：零成本 BFF 到底靠不靠谱？" ✅
- "维护Capa-Java三年，聊聊多运行时架构的那些坑和收获" ✅
- "拿了Hackathon金奖的零成本BFF方案，三年后我还在用" ✅

**标题特点**:
- ✅ 口语化动词或时间词开头
- ✅ 具体时间（"三年/一个月"）
- ✅ 个人经历（"我"）
- ✅ 口头禅（"说实话/聊聊"）
- ✅ 悬念或情绪词（"坑/靠谱"）
- ✅ 无精确数字堆砌

**正文结构**（与 v1.1 一致，经验证效果好）:
```
1. 开头（Hook）：口语化，带情绪，直接入题
2. 项目/问题简介：1段，自然引入
3. 背景/个人经历：2-3段，叙事流
4. 核心内容：3-5段，展开技术/使用细节
5. 优缺点/踩坑：实事求是，不硬吹
6. 适用场景：适合/不适合，增加可信度
7. 代码/配置示例：核心使用片段
8. 总结：口语化感悟
9. 互动引导：轻松提问，引导评论
```

### 模板 B：纯踩坑型

**标题模板**:
```
踩了个大坑：[具体技术问题]
这个问题折磨了我[时间]，记录一下
折腾了[时间]，终于搞定 [问题]
```

**适用场景**: 具体技术问题解决过程

### 模板 C：技术分享型

**标题模板**:
```
[技术] 性能优化实战：从 X 到 X 的经验
深入理解 [技术]：[核心概念]
[项目] 架构设计：我们的实践与反思
```

**适用场景**: 纯技术干货，建立专业影响力

## 🎨 风格指南（v1.2）

### 口语化要求

| 检查项 | 要求 | 示例 |
|--------|------|------|
| 口头禅 | 每篇至少 3 处 | "说实话"、"讲真"、"其实"、"哈哈" |
| 情绪词 | 每篇至少 2 处 | "坑惨了"、"折磨"、"真香"、"崩溃" |
| 具体时间 | 每篇至少 1 处 | "折腾了三年"、"花了一周" |
| 第一人称 | 全程自然使用 | "我发现"、"我觉得"、"我当时" |
| 自嘲/幽默 | 适当使用 | "我自己都笑了"、"说出来你都不信" |

### 反AI检测要求

| 检查项 | 要求 | 说明 |
|--------|------|------|
| 精确数字 | ≤2 个（非技术数据） | 避免"提升 300%"这种堆砌 |
| 段落长度 | 30-150 字混合 | 长短不一，拒绝平均 |
| 结构整齐度 | 不追求完美对齐 | 小标题和段落可混用 |
| 语病/口语 | 适当保留 | "就酱"、"也不是说"、"反正" |
|  emoji/表情 | 少量使用 | 标题和内容都可以有，不要多 |

### 新增建议：增强互动性

在文章末尾增加 1 句轻松的互动引导，可提升评论率：
- "你在 BFF 层踩过哪些坑？评论区聊聊～"
- "有没有类似的经历？欢迎分享你的故事"
- "说实话，你觉得这种方案靠谱吗？"

## 🔄 迭代机制

### 每日检查（cron 自动）

1. 检查 pending 数量
2. 检查 Chrome CDP 端口状态
3. 检查发布日志更新
4. 输出简要报告

### 每周复盘（关键）

待样本达到 10 篇后，启动首次深度复盘：

| 复盘内容 | 输出 |
|----------|------|
| 数据总览 | 阅读量/点赞/评论/收藏汇总 |
| TOP3 文章 | 成功经验总结 |
| BOTTOM3 文章 | 问题诊断 |
| 标题风格验证 | 哪种类型阅读量最高 |
| 策略调整 | 下周计划更新 |

**预计首次有效复盘时间**: 2026-04-24（样本达到 10+ 篇后）
