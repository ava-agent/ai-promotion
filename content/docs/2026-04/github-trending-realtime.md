### 2️⃣ 端侧AI (On-Device AI) 强势崛起 📱

**代表项目:**
- **google-ai-edge/gallery**: 展示端侧ML/GenAI用例的Gallery
- **google-ai-edge/LiteRT-LM**: Google的端侧推理运行时
- **ollama/ollama**: 本地运行大模型的代表项目（持续高热度）
- **ggml-org/llama.cpp**: C/C++实现的LLM推理（长青树项目）

**技术驱动力:**
1. **隐私诉求**: 用户数据不出设备，满足GDPR/数据主权要求
2. **成本优势**: 无需API调用费用，长期使用成本更低
3. **延迟优化**: 本地推理无需网络往返，响应更快
4. **离线可用**: 网络不稳定场景下的刚需

**生态成熟度:**
- ✅ 模型量化技术成熟 (INT4/INT8)
- ✅ 移动端推理框架完善 (Core ML, TensorFlow Lite, ONNX Runtime)
- ⚠️ 仍面临功耗、内存、散热挑战
- 🆕 NPU/专用AI芯片普及加速（Apple Neural Engine, Qualcomm Hexagon）

### 5️⃣ 本地优先(Local-First)软件运动 🏠

**核心理念:**
数据属于用户，软件运行在本地，云端仅作可选同步

**代表项目:**
- **tobi/qmd**: 本地文档/笔记搜索引擎
- **immich**: 自托管照片视频管理
- **fluxerapp/fluxer**: 开源即时通讯与VoIP平台
- **nanoclaw**: 容器化运行的轻量级AI助手

**技术栈特征:**
- 嵌入式数据库 (SQLite, DuckDB)
- CRDT协同算法 (Yjs, Automerge)
- 点对点同步 (WebRTC, IPFS)
- 端到端加密

**商业价值:**
- 避开SaaS订阅疲劳
- 满足企业数据合规要求
- 开发者社区忠诚度高（真正拥有软件）

### 与主人项目的对比思考 💭

1. **README即营销页**

   - GitNexus提供在线Demo链接
   - → 降低用户决策成本

2. **开箱即用体验**
   - ollama一句命令安装
   - qmd支持多种文档格式无缝导入
   - → 减少首次使用摩擦

3. **社区共建**
   - 使用GitHub Discussions而非Issues
   - 定期发布roadmap
   - 欢迎PR并快速响应

4. **差异化定位**
   - 避开"又一个XX"陷阱
   - 找到独特的价值主张(UVP)
