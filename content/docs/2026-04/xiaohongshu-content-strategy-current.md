## 🎨 内容质量标准

### ✅ 质量检查清单
每篇内容必须通过以下检查:
- [ ] 基于真实GitHub项目数据
- [ ] 使用真实stars数和更新时间
- [ ] 无禁用词汇(姐妹们、救命、挖到宝等)
- [ ] 无翻译腔和AI典型特征
- [ ] 具体数字和时间参考
- [ ] Emoji数量控制在5-8个
- [ ] 标签数量3-5个,精准相关
- [ ] 标题15-25字,自然不夸张
- [ ] 优缺点都说,不全是夸
- [ ] 结尾有互动引导

### 🔍 禁用词汇清单
❌ **过度亲密**: "姐妹们"、"宝子们"、"家人们"
❌ **营销套路**: "救命"、"挖到宝了"、"相见恨晚"
❌ **过时流行语**: "绝绝子"、"yyds"、"太香了"、"吹爆"
❌ **推销感**: "快冲"、"闭眼入"、"不看后悔"
❌ **模板化**: "保姆级教程"、"有手就会"
❌ **虚假夸张**: "神仙工具"、"宝藏app"、"效率神器"
❌ **标题党**: "震惊!"、"我不允许还有人不知道"

### 📝 标题公式
**公式1: 数字+结果+情绪**
"测了30+个AI工具,这5个真的好用!建议收藏 ⭐"

**公式2: 悬念+痛点+解决方案**
"做AI旅行规划,我翻车了好几次才搞明白..."

**公式3: 真实体验+反差**
"说实话,这个工具我用了3个月才敢说真话"

**公式4: 时间+问题型**
"做[项目],我被[技术]坑了[时间]"

## 🚀 内容创作优化策略

### 🎯 创作模式

#### 模式A: 踩坑实录 (最受欢迎)
```
开头:场景引入
"最近在做 XX 的 YY 功能,踩了个坑..."

中间:踩坑过程
"本来以为很简单,结果..."
"试了 A,不行"
"试了 B,也不行"
"试了 C,还是不行"

转折:解决方案
"后来发现..."
"原来是这个问题"

结尾:心得 + 互动
"现在回头看..."
"你们有没有遇到过?"
```

#### 模式B: 工具实测
```
开头:为什么用这个工具
"因为当时遇到了 xx 问题..."

中间:真实使用
✅ 好的地方(有具体场景)
❌ 不好的地方(不回避)

数据对比:
| 功能 | 效果 | 备注 |
|------|------|------|

结尾:适合谁 / 不适合谁
"推荐给...人群"
"不太适合...场景"
```

#### 模式C: 创意融合 (新增)
```
开头:跨界灵感
"把XX和YY结合,会怎样?"

中间:创意实现
- 技术如何跨界
- 真实的尝试过程
- 遇到的有趣问题

结尾:感悟邀请
"这个脑洞你觉得怎么样?"
"还有什么创意可以分享?"
```
