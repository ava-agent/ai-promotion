# 用了 MCP 一个月，我的真实感受：这东西真的香

![封面图](https://images.unsplash.com/photo-1516321318423-f06f85e504b3?w=1200)

说实话，一开始我觉得 MCP 就是个噱头。

又是新协议、又是新标准，我已经被各种"下一代技术"搞麻木了。但用了整整一个月之后，我不得不说：**这东西真的香**。

作为一个在携程做跨渠道沟通平台的老码农，我最近半年一直在折腾 AI Coding 和 MCP 工具。今天就跟大家聊聊我的真实踩坑经历，以及 4 个我觉得最值得推荐的 MCP Server。

## 先说说 MCP 是什么（说人话版）

MCP（Model Context Protocol）你可以理解成**AI 的 USB-C 接口**。

以前你想让 Claude 帮你查数据库、发邮件、操作 GitHub，得写一堆胶水代码。现在有了 MCP，AI 直接就能调用这些工具，就像插 U 盘一样简单。

我在团队里推广 MCP 的时候，大家最大的顾虑就是：**"又要学新东西了？"**

但实际上手之后发现，学习成本真的不高。就像你第一次用 Type-C 充电线一样，插上去就行了，不用管背后的协议细节。

## 4 个我每天都在用的 MCP Server

下面这几个是我这一个月用下来，觉得最实用的。都是我真实在项目上跑过的，不是从文档里抄的。

### 1. GitHub MCP Server

这个必须排第一，因为我每天都要查代码、看 PR。

**实际场景**：
- 让 Claude 直接帮我 review PR，告诉有哪些潜在问题
- 查询某个仓库最近一周的 commits，生成变更摘要
- 自动创建 issue，把我发现的 bug 直接丢进去

**代码示例**：
```python
# 让 AI 直接查询 GitHub 仓库信息
# 不需要写任何 API 调用代码，直接说就行
"帮我看看 capa-java 仓库最近一周的 PR 合并情况，重点关注 bugfix 相关的提交"
```

**踩过的坑**：
- 一开始 Token 权限给得太小，很多操作报 403，后来改成有 repo 和 workflow 权限的 classic token 才搞定
- 查询大仓库的时候，AI 容易超时，后来我学会了分页查询

### 2. Feishu MCP Server

我们团队用飞书，这个简直是效率神器。

**实际场景**：
- 让 AI 帮我查未读的飞书消息，生成今日待办
- 自动创建飞书文档，把技术方案草稿丢进去
- 查询群聊里的重要讨论，防止漏掉关键信息

**为什么好用**：以前我每天要花 20 分钟刷飞书，生怕漏了什么重要消息。现在 AI 直接帮我筛选，省下来的时间可以写代码或者摸鱼（不是）。

**踩过的坑**：
- Feishu 的权限管理特别细，搞了好久才明白 app 权限和 user 权限的区别
- 群聊消息太多的时候，AI 会"失忆"，后来我学会了分段查询

### 3. Email MCP Server

这个是我用来做跨渠道沟通的，跟我们业务很相关。

**实际场景**：
- 自动发送技术周报给团队
- 查询重要邮件，根据内容生成任务列表
- 批量发送通知邮件（比如系统告警）

**为什么推荐**：我们平台要对接 SMS、Email、Push、WeChat、LINE、WhatsApp 等 10+ 渠道。Email MCP Server 让我能快速验证邮件渠道的集成逻辑，不用每次都去写测试脚本。

**踩过的坑**：
- Gmail 的 OAuth 配置特别麻烦，后来换了 SMTP 方式才顺利
- 发送 HTML 邮件的时候，样式经常出问题，现在我都用纯文本了

### 4. PostgreSQL MCP Server

数据库查询神器，不用再切终端了。

**实际场景**：
- 直接让 AI 查数据库："帮我看看今天有多少新用户注册"
- 分析慢查询日志，生成优化建议
- 对比测试环境和生产环境的数据差异

**实际代码**：
```sql
-- 让 AI 直接执行查询
-- 不用写 Python 脚本，不用切到 psql
"查询过去 7 天每天的订单量，并按渠道分组，找出增长最快的渠道"
```

**踩过的坑**：
- 一开始没做查询限制，AI 不小心执行了个全表扫描，把数据库搞卡了
- 现在我学会了只给只读权限，而且加上 LIMIT 限制

## 我自己开发的 3 个 MCP Server

上面那些是别人做的，下面说说我基于业务需求自己开发的。

### MCP Video Gen

**用途**：多供应商 AI 视频/语音/音乐生成
**GitHub**：https://github.com/kevinten-ai/mcp-video-gen

**为什么做**：我们平台要做视频内容生成，但供应商太多了（Pika、Runway、HeyGen...），每个 API 都不一样。做了个 MCP Server 统一封装，AI 直接就能调用。

**技术点**：
- 支持多供应商切换
- 统一的参数格式
- 自动失败重试

### MCP 3D Gen

**用途**：多供应商 AI 3D 模型生成
**GitHub**：https://github.com/kevinten-ai/mcp-3d-gen

**为什么做**：Trip Meta 项目需要生成 3D 场景，调研了一圈发现 Meshy、Tripo3D、CSM 各有优劣。用 MCP 封装后，AI 能根据场景自动选择最优供应商。

### MCP Image Gen

**用途**：基于 Google Gemini 的图像生成
**GitHub**：https://github.com/kevinten-ai/mcp-image-gen

**为什么做**：Gemini 的图像生成能力被低估了，而且 API 特别便宜。做了个 MCP Server 让团队能快速用上。

## 优缺点分析（实事求是）

**优点**：
- 真的省时间：以前要写胶水代码，现在直接说就行
- 统一接口：不同工具的调用方式都统一了
- 生态发展快：新 Server 每天都在冒出来

**缺点**：
- 调试麻烦：出错的时候不容易定位是 AI 的问题还是 Server 的问题
- 权限管理复杂：每个 Server 都要配 Token，管理起来头疼
- 稳定性不一：第三方 Server 质量参差不齐

**适合**：经常需要查数据、发消息、操作工具的开发者；想提升效率但不想写大量胶水代码的团队；愿意尝鲜、能接受偶尔踩坑的 early adopter。

**不适合**：对稳定性要求极高的生产环境（再等等）；不想折腾配置、希望开箱即用的同学；对 AI 输出结果要求 100% 准确的场景。

## 这 3 个坑，你一定要避开

**坑 1：权限给太大**

我开始的时候图省事，GitHub Token 给了全权限。结果 AI 不小心给我把某个分支删了，幸好是测试仓库。

**解决方案**：最小权限原则，只给需要的 scope。

**坑 2：没做超时控制**

有次让 AI 查询一个大表，结果卡了 5 分钟都没响应。后来才发现是查询没加 LIMIT。

**解决方案**：给数据库连接设置超时，查询必须加 LIMIT。

**坑 3：盲目相信 AI 的结果**

有次 AI 查数据库告诉我"今天有 1000 个新用户"，我信了。后来人工核对发现是 100 个，AI 把数字看串行了。

**解决方案**：关键数据一定要人工二次确认，特别是涉及业务的数字。

## 快速上手指南

如果你想试试 MCP，这是我的建议路径：

**第 1 周**：装 1-2 个简单的 Server（比如 GitHub），熟悉基本用法

**第 2 周**：尝试用 AI 完成一个实际工作流（比如查代码 + 发消息）

**第 3-4 周**：根据业务需求，考虑要不要自己开发 Server

**必备工具**：
- Claude Desktop（官方支持 MCP）
- Claude Code（命令行版本，适合开发者）
- 一个代码编辑器（Cursor 或者 VS Code）

## 写在最后

MCP 不是银弹，但它确实解决了一个真实问题：**让 AI 能方便地调用各种工具**。

我在携程带团队做跨渠道沟通平台，Token 使用量集团 Top1。这一个月用 MCP 的经验告诉我：**工具的价值在于解决问题，而不是炫技**。

GitHub MCP Server、Feishu MCP Server、Email MCP Server、PostgreSQL MCP Server，这四个是我每天都在用的。如果你也在做类似的事情，不妨试试看。
